# 知树小程序开发方案

> 产品：知树 — 种树 + 看新闻的治愈系微信小程序
> 技术栈：Taro 4.1.9 + React 18 + TypeScript + CSS Modules（遵循 TRAE-generate-mini-app 技能规范）
> 生成方式：通过技能脚本 `init-template.js` 初始化模板，再按七阶段流程开发

---

## 一、方案概要

构建一个双核心玩法的微信小程序：
- **育知**（树木养成）：选择树种种植，浇水/施肥获得经验，幼苗→成树后收获得金币，金币解锁新树种。
- **知讯**（热点新闻）：分领域展示 LLM 中立精简后的 Top10 新闻，阅读新闻获得水分/肥料反哺养成。
- **我的**（个人中心）：展示金币、养成统计、设置。

配套一条独立的**新闻数据管线**（Python + GitHub Actions），每日 9 点 / 21 点抓取 RSSHub + Hacker News，清洗后用 LLM 单条精简，输出 JSON 供小程序读取。

### 核心决策（已与用户确认）
| 决策项 | 选择 |
|--------|------|
| 看新闻获资源触发 | 进入新闻详情页即获得水分+肥料，每日有上限 |
| TabBar 结构 | 3 个 tab：育知、知讯、我的 |
| 新闻点击行为 | 二级详情页展示摘要 + 复制原文链接按钮（规避域名白名单） |
| 数据持久化 | 本地存储（Taro.setStorage），无需云开发（Demo 单设备） |
| LLM 接口 | OpenAI 兼容协议，默认 DeepSeek，通过环境变量配置 |
| 动画 | 本次仅做图片切换，预留 Lottie 接入位 |

## 十二、新闻数据管线（Python + GitHub Actions）

### 12.1 抓取源（`src/news_pipeline/config.py`）
- RSSHub 公共实例（如 `https://rsshub.app`）路由：
  - 澎湃新闻首页 `/pengpai/news` 
  - 知乎热榜 `/zhihu/hotlist`
  - 微博热搜 `/weibo/search/hot`
  - 36 氪 `/36kr/newsflashes`
  - 公众号（按需）
- Hacker News API：`https://hacker-news.firebaseio.com/v0/topstories.json` → 取前 30 条 `item/{id}.json`。
- 兜底静态源：1-2 个公开 RSS（防 RSSHub 失效）。

### 12.2 清洗逻辑（`cleaner.py`，无 LLM）
- 统一字段：标题、来源、发布时间、原文片段、热度分、link。
- 去重：标题相似度（归一化后子串/Jaccard）> 0.8 剔除。
- 过滤：广告/引流/水文关键词黑名单过滤。
- 热度计算：
  - 热搜榜：`score = max(0, 100 - rank*3)`
  - 媒体资讯：`score = 新鲜度加权`（发布时间越近越高）
  - HN：`score = points`（归一化到 0-100）

### 12.3 领域分组（`classifier.py`）
- 预设关键词字典：科技 / 国际 / 国内时政 / 财经 / 社会文娱。
- 按标题+片段关键词命中归类，未命中归"社会文娱"兜底。
- 每域按 `hot_score` 降序取 Top10，其余丢弃。

### 12.4 LLM 单条加工（`llm.py`，OpenAI 兼容）
- 接口：环境变量 `NEWS_LLM_BASE_URL` / `NEWS_LLM_API_KEY` / `NEWS_LLM_MODEL`，默认 DeepSeek（`https://api.deepseek.com`，`deepseek-chat`）。
- 单条独立调用，固定 Prompt（用户给定模板），要求返回纯 JSON `{content, tags}`。
- 解析失败则保留原文片段作为 `show_content`，`tags` 置空，保证不崩。
- 输出最终结构（用户给定）：`source / publish_time / origin_title / show_content / tags / hot_score / link / domain`。

### 12.5 输出（`output.py`）
- 写入 `news-pipeline/data/news.json`，结构同 `NewsData`，含 `updated_at` 与 `domains`。

### 12.6 GitHub Actions（`.github/workflows/news-fetch.yml`）
- 触发：`cron: '0 1 * * *'` 和 `'0 13 * * *'`（UTC = 北京 09:00 / 21:00），支持手动 `workflow_dispatch`。
- 步骤：checkout → setup-python 3.11.9 → 安装 uv → `uv sync` → `uv run python -m news_pipeline` → 提交 `data/news.json` 推送回主分支。
- Secret：`NEWS_LLM_API_KEY`。

- 依赖：`feedparser`、`requests`、`openai`（OpenAI 兼容 SDK）、`pyyaml`、`python-dateutil`。
